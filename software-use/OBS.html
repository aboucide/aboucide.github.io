OBS录屏软件使用教程

OBS软件基础操作指南

一、软件特点与下载安装

1. 软件核心特点

  • 开源免费：无功能限制，无需付费即可使用全部核心功能（录屏、直播、多源采集等）
  • 功能丰富：支持多场景管理、多来源采集、自定义参数调节，适用于视频制作（录屏剪辑）与直播推流
  • 跨平台兼容：支持Windows、macOS、Linux系统

2. 下载渠道

下载渠道 操作说明 适用场景
官网 访问 obs project.com，根据操作系统（Windows/macOS/Linux）选择对应版本下载 推荐首选，确保获取最新官方版本
Steam 打开Steam客户端，搜索“OBS Studio”直接安装 官网下载速度慢或网络不稳定时备选

3. 版本与语言设置

  • 示例版本：以OBS 31.1.1版本为例（不同版本界面逻辑一致，参数位置基本无差异）
  • 语言修改：主界面右侧点击「Settings」（设置）→ 在弹出窗口左侧找到「Language」（语言）→ 选择「简体中文」→ 重启软件生效

二、OBS主界面操作

1. 画面采集设置

画面采集需在主界面左下角「来源」面板点击「+」号添加，核心采集类型如下：

（1）显示器采集（常用）

  • 基础设置：添加后采集方式选择「自动」，若多屏幕需手动选择目标屏幕
  • 画面适配：右键采集源 → 选择「变换」→ 按需调整：
    • 常规场景：选「比例适配屏幕」，确保画面无拉伸
    • 特殊场景（如4:3素材适配16:9画布）：选「拉伸至全屏」

（2）其他常用采集类型

采集类型 用途 关键设置
音频输入采集 采集麦克风声音 添加后在「混音器」手动选择对应麦克风设备
游戏采集 采集特定游戏窗口（非全屏游戏优先用） 默认选择「DXGI」模式，避免掉帧或画面卡顿
浏览器 直接在OBS中加载网页内容（如直播弹幕、网页素材） 输入网页URL，可调整窗口大小
文本/图像 添加静态文字（如直播标题）、图片（如LOGO、背景图） 拖拽调整位置，支持实时修改内容

2. 音轨与混音器

（1）音频添加与控制

  • 添加音频输入采集（麦克风）或音频输出采集（桌面声音）后，在主界面中间「混音器」面板可实时调节音量（拖拽滑块）、静音（点击喇叭图标）

（2）高级音频设置

  1. 点击混音器最下方的「齿轮图标」→ 选择「高级音频属性」
  2. 音轨分配：OBS最多支持6条音轨，建议按用途划分（如：1轨放桌面声音、2轨放麦克风声音），方便后期剪辑时单独控制
  3. 直播音轨设置：直播仅调用1条音轨，需在「高级音频属性」中勾选目标音轨（如勾选1轨），再到「输出设置」中确认对应音轨勾选

3. 画布预览

  • 主界面上方的「画布区域」实时预览当前所有采集源的组合效果，可直接拖拽采集源调整位置、拉伸大小，直观把控最终输出画面

三、核心设置（Settings）

点击主界面右侧「Settings」进入设置窗口，以下为关键设置模块详解：

1. 视频设置（Video）

参数 说明 推荐配置
基础画布分辨率 决定OBS主界面画布的大小与比例，不影响最终输出分辨率 1920×1080（1080P）、3840×2160（4K，需硬件支持）
输出缩放分辨率 实际录制/直播的画面分辨率 通常与「基础画布分辨率」一致；若硬件性能不足，可降为低分辨率（如4K画布→1080P输出）
帧率（FPS） 画面流畅度，数值越高越流畅 默认60FPS；B站等平台4K内容最高支持120FPS（高帧率需更强编码器性能）

2. 输出设置（Output，输出模式设为「高级」）

输出设置分「录制」和「直播」两个标签页，核心参数如下：

（1）通用基础设置

  • 录屏路径：点击右侧「预览」选择视频保存文件夹（建议选择空间充足的磁盘）
  • 录像格式：推荐「混合MP4」（兼容性强，崩溃时大概率能保存文件）；不推荐FLV/MKV（兼容性差，需后期转换）

（2）编码器选择（核心）

场景需求 推荐编码器 编码器类型（软/硬件） 理由 压缩效率对比（vs H.264） 同画质下原H.264为1GB时的估算大小
录屏后要在手机/旧电脑播放 NVIDIA NVENC H.264 硬件编码器 兼容性拉满，所有设备都能打开 基准（100%） 1GB（原文件）
高分辨率（4K/8K）录屏 NVIDIA NVENC HEVC 硬件编码器 压缩效率高，省空间，支持高分辨率+HDR 压缩率高40%（体积为60%） 约0.6GB
想节省存储空间（文件更小） NVIDIA NVENC HEVC/AV1 硬件编码器 HEVC兼容性更好，AV1压缩比更高（需RTX 40系显卡） HEVC：压缩率高40%（体积60%）；AV1：压缩率高60%（体积40%） HEVC约0.6GB；AV1约0.4GB
追求极致画质（不赶时间） x264 软件编码器 软件编码算法更精细，相同码率下画质比硬件H.264更好 画质更优但体积相近 约1GB（同H.264）
长期存储/低带宽分享 NVENC AV1（有RTX 40系） 硬件编码器 压缩比最高，未来兼容性会逐步普及（没新显卡可选HEVC） 压缩率高60%（体积为40%） 约0.4GB（HEVC替代时约0.6GB）

注：

  • 硬件编码器依赖显卡芯片加速（如NVIDIA的NVENC模块），编码速度快、CPU占用低；
  • 软件编码器依赖CPU运算，编码速度慢、CPU占用高，但算法优化更精细，同参数下画质可能更优。
  • 实际文件大小受分辨率、帧率、码率设置影响，压缩效率为同画质下的理论对比值。

（3）比特率控制（关键参数，影响画质与文件大小）

优化后的比特率控制参数表格（修正细节+补充场景适配）：

控制模式 原理 适用场景 推荐设置
CBR（恒定比特率） 码率固定，不随画面复杂度变化 直播（保证推流稳定，避免观众卡顿） 1080P直播：4000-6000 Kbps；4K直播：10000-15000 Kbps
CQP（恒定QP） 按画面质量自动调整码率（质量等级越低，码率越高） 录屏（保证画质，避免静止画面浪费码率） 质量等级23-28（23：画质清晰，文件较大；28：画质平衡，文件适中）
VBR/CQVBR VBR：画面复杂时升码率、简单时降码率；CQVBR：恒定质量+可变码率结合 录屏（画质与文件体积平衡，比CBR更灵活） 1080P录屏：VBR上限8000 Kbps；CQVBR质量等级25+码率上限10000 Kbps
无损录制 仅支持英伟达显卡，通过CQP=0/1实现无压缩录制 专业级画面需求（如视频后期精细剪辑，需保留原始画质） CQP=0（完全无损，文件体积极大，需充足存储+高性能显卡）

（4）其他关键参数

  • 质量等级/预设：数字越小，画质越高、文件越大、越吃性能；若编码器卡顿，可适当提高数值（如从「Quality 1」改为「Quality 3」）
  • 配置文件：选「High」（文件小、清晰度高），不推荐Main/Baseline（画质差）
  • 关键帧间隔：设2秒（符合直播平台要求，剪辑时画面衔接更流畅）
  • B帧：直播设2帧，录屏（显卡支持）设3-4帧（减少文件大小，不影响画质）
  • 重新缩放输出：需同时录屏+直播且分辨率不同时开启（如录屏4K，直播1080P）
  • 音频编码器：默认「AAC」（非专业音频无需修改）；音频码率最高320 KPPS，可自定义音轨名称（如“麦克风音轨”）

3. 直播设置（Stream）

  1. 服务选择：在「Service」下拉框中点击「显示全部」，找到目标直播平台（如Bilibili、Twitch）
  2. 服务器：选「全球」或对应地区服务器（如“中国-华东”），确保推流稳定
  3. 推流码获取与填写：点击「获得推流码」，按平台提示（如登录账号、创建直播间）复制「推流地址」和「串流密钥」，粘贴到OBS对应位置

4. 回放缓存设置（Replay Buffer）

  • 功能：硬盘空间不足时，临时缓存高画质精彩瞬间（如游戏击杀、直播名场面），需手动触发保存
  • 操作设置：
    • 勾选「启用回放缓存」，设置缓存时间（如30秒，即缓存最近30秒画面）
    • CQP等非固定码率模式需设置「最大内存使用量」（默认512MB，可根据内存大小调整，建议测试稳定性）

5. 高级设置（Advanced）

  • 进程优先级：若游戏占满性能导致录屏卡顿，将OBS进程优先级设为「高于正常」或「高」
  • 色彩范围：非HDR场景选「常规（limited）」，HDR场景需测试「常规」或「扩展（full）」（避免画面泛白/偏色）
  • 录制保护：若选MKV/FLV格式（崩溃时文件不损坏），可勾选「自动封装为MP4」（录制结束后自动转MP4）；可勾选「覆盖旧文件」（避免重复文件名冲突）
  • 直播延迟：按需调整（如“低延迟”适合互动直播，“高延迟”适合稳定推流）
  • 快捷键：不建议给「开始直播/录制」设快捷键（避免未触发或误操作，导致漏录/误开播）

四、常见问题解决

1. 显示器采集问题

（1）黄边/黑屏

  • 原因：游戏与OBS调用的显卡不同（如游戏用核显，OBS用独显）
  • 解决步骤：
    1. 打开系统「显示设置」→ 点击「图形设置」→ 选择「浏览」，添加OBS程序（官网下载在C:\Program Files\obs-studio\bin\64bit\obs64.exe；Steam下载在C:\Program Files (x86)\Steam\steamapps\common\OBS Studio\bin\64bit\obs64.exe）
    2. 为OBS设置「高性能GPU」（独立显卡），开启硬件加速
    3. 重启OBS，若无效可换「游戏采集」模式

（2）帧率掉/帧生成时间差

  • 解决：在显示器采集设置中，将「采集方式」从「Windows 10采集」改为「DXGI截屏」，减少资源占用

2. 画布色彩问题（泛白/颜色异常）

  • 解决：进入「高级设置」→ 找到「色彩范围」→ 非HDR场景切换为「常规（limited）」，HDR场景测试「扩展（full）」，重启OBS后查看效果
